# New User Feature Organization

## 60-second mental model
1. **Scan or add schoolwork** — syllabus/photo/manual entry creates classes and assignments.
2. **Review what was found** — user confirms dates and duplicates before trusting the plan.
3. **See today and the week** — Today answers “what now?”; Plan answers “what is coming?”
4. **Manage classes** — classes are containers for assignments, meetings, grades, and color identity.
5. **Customize widgets/reminders** — home screen and reminders keep the plan visible.
6. **Upgrade only when expanding** — Plus unlocks more imports, automation, focus/grades, and advanced customization.

## Recommended tabs
- **Today**: action dashboard, next deadlines, quick homework, demo path.
- **Scan**: import syllabus/photo/typed homework, parse/review handoff.
- **Plan**: week and semester timeline.
- **Classes**: courses, assignments by class, meetings, manual additions.
- **Widgets**: Widget Studio, app themes, native widget install state.
- **More/Plus**: currently folded into Widgets + Upgrade; future split only if settings grow.

## Progressive disclosure
- First launch: one promise, one CTA, one demo escape hatch.
- Empty Today: explain Scan/Add/Demo.
- Widget Studio empty state: previews clearly say “add homework to preview” instead of pretending.
- Plus gates: show what remains free and what is advanced.
